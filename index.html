CSS Toggle Switches

Accessible, CSS-only, toggle switches with mobile support, using standard form elements.

Update: The latest version does not require JavaScript for mobile support.

You can read more about the development process:

  • CSS Toggle Switches
  • Mobile support for the CSS toggle switches

Checkbox

For "On/Off" states, checkboxes are best used. 


<fieldset class="toggle">
	<input id="view" type="checkbox" checked />
	<label for="view" onclick="">Wireless:</label>
	<span class="toggle-button"></span>
</fieldset>

Radio

If you need to different states that are more specific than "On/Off", or more than two of them, the best approach is to use radio buttons. 


<fieldset class="switch">
	<legend>View: </legend>
	
	<input id="week" name="view" type="radio" checked>
	<label for="week" onclick="">Week</label>

	<input id="month" name="view" type="radio">	
	<label for="month" onclick="">Month</label>
	
	<span class="switch-button"></span>
</fieldset>

Multiple states

The radio button styles also include an additional layout, for tri-state switches, by adding an additional switch-treo class to the fieldset.

You can create your own styles for switches with multiple states, by following the .switch-treo example, in the CSS file. 


<fieldset class="switch switch-treo">
	<legend>View: </legend>
	
	<input id="week4" name="view4" type="radio" checked>		
	<label for="week4" onclick="">Week</label>

	<input id="month4" name="view4" type="radio">
	<label for="month4" onclick="">Month</label>
	
	<input id="year4" name="view4" type="radio">
	<label for="year4" onclick="">Year</label>
	
	<span class="switch-button"></span>
</fieldset>

Compatibility

The toggle switches should work on all modern browsers, including mobile ones.

Browsers without support for mediaqueries, such as IE8 and bellow, get standard form elements.

The onclick="" attribute is required for older iOS and Opera Mini support.
